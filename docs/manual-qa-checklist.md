# רשימת QA ידנית - ששבש טורקי אונליין

מסמך זה משמש כרשימת בדיקות חובה לפני ואחרי כל שינוי בפרויקט. המטרה היא לוודא שהמשחק הקיים לא נשבר, במיוחד סביב החוקים המיוחדים והבאגים שחזרו בעבר.

## כללי בדיקה

- לבדוק בדפדפן רגיל לפני כל שינוי משמעותי.
- לבדוק שוב אחרי כל שינוי.
- באונליין לבדוק עם שני חלונות, שני טאבים או שני מכשירים.
- לא לאשר שינוי אם אחד מהבאגים החוזרים מופיע שוב.
- אם שינוי נוגע ללוגיקה, לבדוק גם משחק מול מחשב וגם אונליין.
- אם שינוי נוגע ל-UI, לבדוק גם מסך דסקטופ וגם מסך מובייל צר.

## בדיקת טעינה בסיסית

- [ ] המשחק נטען בלי שגיאות בקונסול.
- [ ] מסך החשבון הראשון קומפקטי ומציג רק כותרת, סטטוס חשבון, "המשך כאורח" ו-Google.
- [ ] אחרי "המשך כאורח" מופיע שדה קצר "איך לקרוא לך?"; אם משאירים ריק נשאר fallback אורח.
- [ ] בחירת המשחק מופיעה רק אחרי המשך כאורח / חיבור Google / חשבון Google שכבר מחובר.
- [ ] כפתור Google במסך הפתיחה לא נראה שבור: אם Firebase/דומיין לא מוגדרים, מופיעה הודעה ידידותית שאפשר להמשיך כאורח.
- [ ] אם Google כבר מחובר, מסך הבית מציג "חשבון Google מחובר" / "מחובר עם Google" ולא מציג פעולת חיבור מבלבלת.
- [ ] הקישור "הפרופיל שלי" מופיע במסך בחירת המשחק ופותח את פאנל הפרופיל הקיים.
- [ ] הכפתורים "שחק מול מחשב" ו"שחק מול חבר" ברורים ומעבירים לפאנלים הקיימים, בלי בחירת גלגול במסך הפתיחה.
- [ ] כפתור גלגול ידני/אוטומטי הקטן מופיע בתוך אזור פעולות המשחק.
- [ ] בתוך המשחק, "חוקים מיוחדים" מוצג כברירת מחדל ככפתור קטן בלבד; לחיצה פותחת וסוגרת את רשימת החוקים.
- [ ] בתוך המשחק, "לוג מהלכים" מוצג כברירת מחדל ככפתור קטן בלבד; לחיצה פותחת וסוגרת את הלוג.
- [ ] במסך בחירת המשחק, "שחק מול מחשב" ו"שחק מול חבר" הם הכפתורים המרכזיים, ו"חוקים" נראה משני וקטן יותר.
- [ ] בוחר האווטארים מציג כמה אפשרויות מומלצות וכפתור "עוד" שמרחיב לאווטארים מובנים נוספים בלבד.
- [ ] הערת ההתקדמות נשארת הבטחת "בקרוב" בלבד — בלי מטבעות/XP/סטטיסטיקות אמיתיים.
- [ ] הלובי נפתח בעברית ובכיוון RTL.
- [ ] קיימות אפשרויות משחק מול מחשב, משחק מול חבר וחוקים.
- [ ] הלוח מוצג בצורה תקינה.
- [ ] מגשי `White off` ו-`Black off` נמצאים מחוץ ללוח בצד שמאל.
- [ ] פאנל החוקים ופאנל המידע נכנסים למסך.
- [ ] כפתורי גלגול, Undo, End Turn ומשחק חדש מוצגים.

## Undo

- [ ] אחרי מהלך אחד בתוך תור, כפתור Undo נשאר זמין.
- [ ] Undo מחזיר את הלוח למצב הקודם.
- [ ] Undo מחזיר גם remaining moves, dice, bar ו-off.
- [ ] Undo לא נעלם עד שכל התור מסתיים או עד שהתור נגמר רשמית.
- [ ] אחרי סוף תור, Undo מושבת.
- [ ] Undo לא זמין בזמן תור היריב או תור המחשב.
- [ ] Undo לא גורם לסנכרון אונליין שגוי.

## End Turn

- [ ] אי אפשר ללחוץ End Turn כשעדיין יש מהלכים חוקיים זמינים.

- [ ] אם יש מהלך חוקי אחד לפחות, End Turn נשאר מושבת.
- [ ] אחרי End Turn התור עובר לשחקן הנכון.
- [ ] בדאבל רגיל, סוף ארבעת המהלכים נותן תור נוסף לאותו שחקן.
- [ ] ב-`4-5`, סוף ארבעת המהלכים לא נותן תור נוסף.

## בדיקות ידניות אחרי פריסת Firebase Functions (Trusted Stats)

- [ ] האפליקציה עדיין נטענת ללא שגיאות קריטיות.
- [ ] "הפרופיל שלי" עדיין נפתח ועובד.
- [ ] חדר אונליין עדיין נפתח, הצטרפות עדיין עובדת, והמשחק מתחיל תקין.
- [ ] (מומלץ) לסיים משחק בדיקה אונליין ולוודא שנוצר submission חדש.
- [ ] ב-`matchResultSubmissions/{uid}/{matchId}` נכתב `serverReview` על ידי השרת.
- [ ] ב-`trustedStatsApplications/{matchId}` נרשמת החלה/דופליקט (idempotency).
- [ ] ב-`playerStats/{uid}` מתעדכנים נתונים רק אחרי פעולת Function.
- [ ] אין כתיבה ישירה מהקליינט ל-`playerStats`.
- [ ] אי אפשר מהקליינט לכתוב `serverVerified=true` או `trustedStatsApplied=true`.

## תנאי אישור שינוי

שינוי נחשב בטוח רק אם:

- [ ] המשחק עדיין נטען.
- [ ] משחק מול מחשב עובד.
- [ ] אונליין עובד בשני חלונות.
- [ ] Undo עובד לפי הכללים.
- [ ] End Turn לא מאפשר סיום מוקדם.
- [ ] כל החוקים המיוחדים עובדים.
- [ ] ירושלמי עובד לשני הצדדים.
- [ ] אין חזרה של באגים ידועים.
- [ ] לא נגעו בקבצים שלא היו חלק מהמשימה.

## בדיקות בסיס לפונקציית אימות תוצאות (Backend)

- [ ] אם Functions לא נפרסו עדיין, המשחק עדיין נטען ועובד רגיל ללא קריסה.
- [ ] הלקוח עדיין לא יכול לכתוב ישירות `playerStats/{uid}`.
- [ ] הלקוח עדיין לא יכול ליצור submission עם `serverVerified=true` או `trustedStatsApplied=true`.
- [ ] אחרי deploy לפונקציה: submission חוקי יוביל לעדכון trusted stats לשני השחקנים.
- [ ] אותו `matchId` לא נספר פעמיים (idempotency עובד).
- [ ] לא נוצרים/נכתבים שדות coins/XP/rewards בשום נתיב trusted.

## בדיקות דיאגנוסטיקה בטוחות ל-Trusted Stats (ללא כלי אדמין באפליקציה)

- [ ] אחרי סיום משחק אונליין בדיקה, לזהות `matchId` אחרון דרך `matchResultSubmissions/{uid}` ב-Firebase Console.
- [ ] לוודא שבאותו submission יש `serverReview` שנכתב ע"י השרת.
- [ ] לוודא שבפרופיל/רענון סטטיסטיקות מתקבל `hasTrustedStats=true` אחרי החלה מוצלחת (או `false` אם עדיין אין נתונים).
- [ ] לוודא שרענון סטטיסטיקות בפרופיל מציג מספרים מעודכנים רק אחרי שה-Function החיל תוצאה.
- [ ] לוודא שלא נוסף שום כפתור כתיבה מסוכן ל-UI.
